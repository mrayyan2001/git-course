Git & GitHub Full Course

Learn Version Control and Collaboration with Git & GitHub

What is Git?

Git is a free, open-source distributed version control system.

Tracks changes in source code during software development.

Helps manage collaboration among teams.

Why Use Git?

Enables multiple developers to work on the same project simultaneously.

Tracks who made changes, why, and when.

Allows reverting to previous versions if needed.

Installing Git

Download from [https://git-scm.com/](https://git-scm.com/).

Install using package managers:

  • Windows: winget install -e --id Git.Git
  • macOS: brew install git
  • Linux: sudo apt-get install git

Check installation: git --version

Configuring Git

Three levels of configuration:

  • System-Level (--system)
  • Global-Level (--global)
  • Local-Level (--local)

Commands:

  • git config --global user.name "Your Name"
  • git config --global user.email "your.email@example.com"
  • git config --list

Git Initialization

git init: Initializes a Git repository.

.git directory stores version history.

git rm .git/ -rf: Removes the .git directory.

Git Status

git status: Shows untracked, modified, and staged files.

Git Add

Workflow: Working Area → Staging Area → Repository

Commands:

  • git add : Adds a specific file.
  • git add .: Adds all files (not recommended).
  • git reset : Removes a file from staging.

Git Ignore

.gitignore: Specifies files/directories to ignore.

Example:

        # Ignore log files
        *.log

Git Commit

git commit -m "Message": Saves changes to the repository.

Best Practices:

  • Be selective about what to commit.
  • Write clear commit messages.

The Perfect Commit

Selective changes: Focus on one topic per commit.

Good commit message:

  • Subject: Concise summary (<80 characters).
  • Body: Detailed explanation.

Git Log

git log: Shows commit history.

Useful flags:

  • --oneline: Condensed view.
  • --graph: Graphical view.
  • --all: Show all branches.

Git Alias

Create shortcuts for Git commands:

  • git config --global alias.lg "log --oneline --graph --all"

Git Remote

git remote: Manages connections to remote repositories.

Example:

  • git remote add origin
  • git remote -v: Lists remotes.

Git Push

git push -u origin main: Sends changes to the remote repository.

Avoid force-pushing (-f).

Git Pull

git pull: Fetches and merges changes from the remote repository.

git pull = git fetch + git merge

Git Clone

git clone : Copies a remote repository to your local machine.

GitHub Codespaces

Online development environment in your browser.

Steps to create a Codespace.

Git Branches

git branch: Lists branches.

git branch : Creates a new branch.

git branch -d : Deletes a branch.

Git Checkout

git checkout : Switches branches.

git checkout -b : Creates and switches to a new branch.

Git Merge

git merge : Combines changes from another branch.

Resolving merge conflicts manually.

Feature Branch Workflow

Steps:

  1. Create a feature branch.
  2. Develop and commit changes.
  3. Push to GitHub and create a pull request.
  4. Review and merge.

Branching Strategies

Long-running branches: main, develop.

Short-lived branches: feature, bugfix.

Popular strategies: GitHub Flow, Git Flow.

Pull Requests

Facilitate code review and collaboration.

Steps to create a pull request on GitHub.

Merge vs Rebase

Merge: Combines branches with a new commit.

Rebase: Rewrites history for a linear timeline.

Stash

git stash: Temporarily saves changes.

Commands: stash, stash list, stash pop.

Interactive Rebase

Modify commit history:

  • Change messages.
  • Combine commits.
  • Delete commits.

Cherry-Pick

git cherry-pick : Copies a specific commit to another branch.

Reflog

git reflog: Logs every movement of HEAD.

Useful for undoing mistakes.

Revert

git revert : Safely undoes changes.

Submodules

git submodule add : Includes external repositories.

Search and Find

Filter commit history:

  • By date, author, message, or file.
